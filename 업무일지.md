### 오늘 작업 내역

#### 1. 프로젝트 복구 기능
- `deletedProjectIds[]` 소프트 삭제 → `restoreProject()` 액션 추가
- 삭제된 항목 있을 시 상단 바에 `🗑 삭제된 항목 N건` 버튼 표시
- 클릭 시 하단 복구 패널 펼침, 개별 버튼으로 복구

#### 2. 프로젝트명/발주처 수정
- `projectMetaEdits: Record<string, {projectName, client}>` 스토어 추가 (localStorage 영속)
- `updateProjectMeta()` 액션 추가
- `useFilteredProjects`에서 수정 내역 반영
- 간트 행 호버 시 ✏️ 버튼 → Modal로 편집 (createPortal, overflow 영향 없음)

#### 3. 유사 중복 경고 개선
- `duplicateConflicts: Map<id, [{id, text}]>` 구조로 변경 (충돌 프로젝트 ID 포함)
- ⚠️ 클릭 시 클릭 위치 근처에 충돌 목록 팝업 (createPortal)
- 충돌 프로젝트 클릭 → 스크롤 이동 + 2.5초 노란색 하이라이트

#### 4. 수정/삭제 버튼 호버 안정화
- JS 상태(`hoveredRowId`) 방식 제거
- CSS named-group 방식도 불안정 → `index.css` 순수 CSS (`.name-cell:hover .row-actions`) 로 최종 교체
- 프로젝트명 셀에 마우스 올릴 때만 표시, 벗어나면 즉시 사라짐 (완전 해결)

#### 5. A4 출력 최적화
- `@page { size: A4 landscape; margin: 6mm 5mm; }`
- 출력 기본 폰트 13pt (`body { font-size: 13pt }`)
- 표 헤더 10pt, 주차 셀 7-8pt
- 컬럼 너비 px 고정 → % 비율로 변경 (A4 전체 너비 자동 채움)

#### 7. Claude Code 텔레그램 MCP 서버 오류 해결 (2026-03-23)
- **증상**: Claude Code 우측 상단 "1 MCP server failed" 오류, 텔레그램 미작동
- **원인**: 텔레그램 플러그인이 `bun` 런타임 사용 → `bun`이 설치되어 있으나 `~/.zprofile` PATH에 미등록
- **해결**: `~/.zprofile`에 `export PATH="$HOME/.bun/bin:$PATH"` 추가 → Claude Code 재시작 필요

#### 8. 텔레그램 봇 페어링 및 접근 설정 문제 해결 (2026-03-23)
- **증상**: `/start` 메시지는 수신되나 봇에 메시지 보내도 응답 없음
- **원인 1**: `access.json`의 `dmPolicy`가 `allowlist`로 설정되어 있어 미등록 사용자 메시지 차단
- **원인 2**: allowlist에 `8745699330`(봇 자체 ID) 잘못 등록됨, 실제 사용자 ID는 `8152882784`
- **해결 과정**:
  1. `dmPolicy`를 `allowlist` → `pairing`으로 변경
  2. 텔레그램에서 메시지 전송 → 6자리 코드(`6eb252`) 수신
  3. `/telegram:access pair 6eb252` 실행 → 사용자 ID `8152882784` allowlist 등록
  4. `dmPolicy`를 `pairing` → `allowlist`로 재변경 (보안 잠금)
- **현재 상태**: 봇 페어링 완료, 텔레그램 메시지 → Claude Code 세션 전달 동작 중
